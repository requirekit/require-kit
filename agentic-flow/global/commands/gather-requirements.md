# Gather Requirements Command

Start an interactive requirements gathering session using the Agentecflow methodology.

## Usage
```
/gather-requirements [feature-name]
```

## Process

I'll guide you through a structured Q&A session following the Agentecflow requirements methodology:

### Phase 1: Discovery
- Understanding the problem
- Identifying users and stakeholders
- Defining success criteria

### Phase 2: Exploration  
- Detailing functionality
- Exploring scenarios
- Identifying constraints

### Phase 3: Validation
- Confirming completeness
- Ensuring clarity
- Assessing feasibility

## What You'll Get

After our session:
1. Natural language requirements documented
2. Clear acceptance criteria defined
3. Test scenarios identified
4. Ready for EARS formalization

## Reference

This command uses the methodology from:
`~/.agentic-flow/instructions/core/requirements-gathering.md`

Start by telling me: **What feature or capability would you like to explore?**
